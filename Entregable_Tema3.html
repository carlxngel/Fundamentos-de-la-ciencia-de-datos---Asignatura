Entregable: Loan Approval Classification

Descarga el archivo: loan_data.csv donde se incluyen datos ficticios asociados a clientes que han solicitado crdito a una organizacin financiera. La descripcin de las columnas del dataset es la que sigue:

  • person_age: Edad
  • person_gender: Gnero
  • person_education: Mayor nivel educacional
  • person_income: Ingreso anual
  • person_emp_exp: Aos de experiencia laboral
  • person_home_ownership: Estado actual de vivienda (e.g., rent, own, mortgage)
  • loan_amnt: Monto solicitado de crdito
  • loan_intent: Objetivo del crdito
  • loan_int_rate: Tasa de inters del crdito
  • loan_percent_income: % del crdito respecto al ingreso anual
  • cb_person_cred_hist_length: Aos de historia crediticia
  • credit_score: Credit score de la persona
  • previous_loan_defaults_on_file: Indicador de solicitudes de crdito previas
  • loan_status: (target variable) Loan approval status: 1 = approved; 0 = rejected

Source: https://www.kaggle.com/datasets/taweilo/loan-approval-classification-data/data

Las variables categricas principales son:

  • person_gender
  • person_education

  • previous_loan_defaults_on_file

Las variables numricas incluyen:

# Aplicar Winsorizing (limitar a percentiles P1-P99)
winsorize_fun <- function(x, p_lower = 0.01, p_upper = 0.99) {
  lower <- quantile(x, p_lower, na.rm = TRUE)
  upper <- quantile(x, p_upper, na.rm = TRUE)
  pmin(pmax(x, lower), upper)
}

1. Estructura Inicial del Dataset

  • Tamao original: 45,000 registros y 14 columnas
  • Tipo de variables: 8 numricas, 6 categricas (incluyendo target)
  • Datos faltantes: 0 (dataset completamente integrado)

Se realiz un anlisis exhaustivo de outliers utilizando el mtodo IQR (Rango Intercuartlico), identificando:

3. Estrategia de Limpieza Implementada

Se aplic una estrategia mixta de limpieza segn la naturaleza de cada anomala:

A. Eliminacin de Registros (Errores Claros)

  • person_age > 100: Biolgicamente implausible. Max encontrado: 144 aos
  • person_emp_exp > 50: Poco realista en contexto laboral. Max encontrado: 125 aos
  • Total eliminados: 2,332 registros (5.18% del dataset)

B. Winsorizing (Limitacin a Percentiles P1-P99) Aplicado a variables con valores extremos pero potencialmente reales:

  • person_income: Limitado a rango [17,875, 271,450] (vs mximo original 7.2M)
  • loan_amnt: Limitado a rango [1,138, 28,390]
  • cb_person_cred_hist_length: Limitado a rango [2, 17]

4. Impacto de la Limpieza

Dataset Original: 45,000 registros Dataset Limpio: 42,668 registros Prdida de datos: 2,332 registros (5.18%)

Estabilidad de la Variable Target (loan_status):

  • Original: 32,850 aprobados | 12,150 rechazados (73.0% vs 27.0%)
  • Limpio: 31,590 aprobados | 11,078 rechazados (73.9% vs 26.1%)
  • Conclusin: La distribucin se mantiene equilibrada despus de la limpieza

X_all <- data.frame(num_scaled, dummies)
y_all <- df_model$loan_status

# Divisin estratificada 70/30
train_idx <- unlist(tapply(seq_len(nrow(df_model)), y_all, function(idx) {

}))

train_df <- cbind(loan_status = y_all[train_idx], X_all[train_idx, ])

# Predicciones en test
